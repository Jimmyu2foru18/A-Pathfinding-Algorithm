Path Reconstruction

The Final Step: Building the Path

Once A* reaches the goal node, the search is complete. However, we only have a chain of parent pointers leading backward from the goal to the start. The final step is to reconstruct the usable path.

Basic Reconstruction Algorithm

We trace the path by starting at the goal and following the parent reference until we reach the start node (which has a null parent).

function reconstructPath(goalNode) {
    const path = [];
    let current = goalNode;
    
    while (current !== null) {
        path.push(current);
        current = current.parent;
    }
    
    return path.reverse();  // Reverse to get Start to Goal order
}

Advanced Techniques

1. Coordinate-Only Storage

To save memory, instead of storing full parent objects, store only the parentX and parentY coordinates in each node.

2. Direction-Based Storage

Store an enum or integer representing the direction taken to reach a node (e.g., UP, DOWN, LEFT, RIGHT). This can significantly reduce memory usage for large grids.

Path Smoothing

Paths generated by grid-based A* can often be "jagged." Line-of-sight algorithms can be used to skip intermediate nodes that aren't necessary for a direct path, effectively smoothing the result.
