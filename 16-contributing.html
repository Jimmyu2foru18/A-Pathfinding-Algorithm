Contributing Guide

Welcome Contributors!

Thank you for your interest in contributing to this repository! We welcome bug fixes, documentation improvements, new features, and research insights.

How to Contribute

  1. Fork and Clone: Create your own fork and clone the repository locally.
  2. Create a Branch: Work on your changes in a dedicated branch.
  3. Develop: Implement your changes, following our code standards.
  4. Test: Ensure all changes are thoroughly tested.
  5. Submit a Pull Request: Open a PR with a clear description of your changes.

Code Standards

Please adhere to our style guidelines to maintain project consistency:

  • Use descriptive camelCase for functions and variables.
  • Document all public functions with JSDoc.
  • Keep functions focused and modular.
